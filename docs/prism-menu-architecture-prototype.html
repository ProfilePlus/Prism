# Prism 菜单重构提案

核心原则：顶部菜单负责完整命令，右键菜单负责对象上下文，状态栏只保留状态和高频动作。

选中文本后，编辑区右键只显示真实可用的基础格式：粗体、斜体、下划线、删除线。

预览区右键不混入编辑动作，专注于复制、定位和导出。

Prism 菜单信息架构

目标是让每个入口只承担一种责任：全局命令、视图切换、对象操作、文本操作、阅读操作彼此分离。

重构原则

  • 不展示未实现的功能，不用“即将推出”污染主菜单。
  • 顶部菜单完整但低频，标题栏和状态栏只放高频动作。
  • 右键菜单跟随对象变化，不做通用功能垃圾桶。
  • 编辑区和预览区的右键菜单职责完全不同。

入口职责

顶部菜单完整命令树、键盘用户、低频但确定可用的能力。 标题栏编辑、分栏、预览三种主视图。 状态栏保存状态、字数行列、侧边栏、专注、导出入口。 文件树文件、目录、工作区对象操作。 编辑区剪贴板和基础行内格式。 预览区复制、源码定位、导出、链接操作。
