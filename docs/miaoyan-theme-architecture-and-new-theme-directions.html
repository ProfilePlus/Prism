MiaoYan Compatibility Mode

当前 miaoyan 不是普通配色，而是一套独立 compatibility mode。新主题要沿用同一个结构：独立 DOM、独立 Markdown pipeline 分支、独立编辑器染色、独立 Mermaid 配置。

架构概览

XBlog 采用“前端 + 后端 + 数据库 + 对象存储 + 反向代理”的标准拆分方式。公网访问统一从 Caddy 进入，内部服务彼此通过 Docker 网络通信。

  • apps/admin：管理后台
  • apps/api：后端服务
  • packages/contracts：前后端共享的数据契约
每个新主题都应该像 miaoyan 一样自带完整 Markdown 排印规则，而不是复用通用预览基座。

1. Theme Registry

把 miaoyan 的判断抽象成 compatibility registry，但每个主题仍输出自己的 DOM 和 CSS，不共享排印。

2. Editor Mode

每个新主题有独立 CodeMirror 参数：字体、字号、行高、语法染色、是否深色。

3. Preview Mode

每个新主题拥有 .preview-compat--name，独立处理 h1、列表、表格、代码、引用、图片。

4. Mermaid Mode

每个主题自带 Mermaid themeVariables、字体加载等待、viewBox/overflow 修正。
