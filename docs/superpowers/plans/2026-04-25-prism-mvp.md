# Prism Markdown 编辑器实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 从零搭建 Prism Markdown 桌面编辑器，基于 Tauri 2 + React 18 + TypeScript，实现文件打开、多标签页、编辑/预览模式、自动保存、主题切换等核心功能。

**Architecture:** 采用文档中心架构，按领域拆分（workspace / document / editor / export / settings），使用 Zustand 管理状态，CodeMirror 6 作为编辑器，unified 渲染 Markdown。

**Tech Stack:** Tauri 2.x, React 18, TypeScript, Vite, Zustand, CodeMirror 6, unified (remark + rehype), KaTeX, Mermaid

import { useSettingsStore } from '../store';

export function ThemeToggle() {
  const { theme, setTheme } = useSettingsStore();

  const toggleTheme = () => {
    setTheme(theme === 'light' ? 'dark' : 'light');
  };

  return (
    <button
      onClick={toggleTheme}
      style={{
        padding: '8px 16px',
        borderRadius: '6px',
        border: '1px solid rgba(127, 127, 127, 0.3)',
        background: 'transparent',
        cursor: 'pointer',
        fontSize: '14px',
      }}
    >
      {theme === 'light' ? '🌙 深色' : '☀️ 浅色'}
    </button>
  );
}

创建 `src-tauri/src/commands/fs.rs`：

```rust
use tauri::command;
use std::fs;

#[command]
pub async fn open_file_dialog() -> Result<Option<String>, String> {
    use tauri::api::dialog::blocking::FileDialogBuilder;
    
    let file = FileDialogBuilder::new()
        .add_filter("Markdown", &["md", "markdown"])
        .pick_file();
    
    Ok(file.map(|p| p.to_string_lossy().to_string()))
}

#[command]
pub async fn read_file_content(path: String) -> Result<String, String> {
    fs::read_to_string(&path)
        .map_err(|e| format!("Failed to read file: {}", e))
}

#[command]
pub async fn write_file_content(path: String, content: String) -> Result<(), String> {
    fs::write(&path, content)
        .map_err(|e| format!("Failed to write file: {}", e))
}
```
